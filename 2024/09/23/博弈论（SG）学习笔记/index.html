2 前置知识

2.1 NP 状态介绍

NP 状态，四舍五入为当前必胜态和当前必败态。引入情境：

桌子上有一包饼干，Pretharp 和 Mijacs 轮流选择吃掉还是不吃掉，吃掉饼干的获得胜利。Pretharp 先手，当双方都采用最优状态时，问输赢。

很显然，一开始这个问题的状态就是必胜态，因为 Pretharp 只要吃掉饼干就赢了，所以一开始状态就是 N 状态。那么对于 Mijacs 来说呢？很显然是一个必败态，也就是 P 状态，因为 Pretharp 不是傻子。

这样手动推理出 NP 状态并不难，可是如果我们将问题拓展：

桌子上有 nnn 包饼干，Pretharp 和 Mijacs 轮流选择吃掉一包还是两包，如果当前没得饼干吃了则输掉游戏。Pretharp 先手，当双方都采用最优状态时，问输赢。

先考虑没有饼干的状态，这很显然是一个必败态。那么如果当前还剩下一包或者两包饼干，这则是一个必胜态。证明都很好证明，那么我们怎么用组合游戏的语言去描述当前状态是必胜态还是必败态呢？因为对于还剩下一包饼干或者两包饼干时，当前操作者都可以使用一次操作将操作变为必败态，这个必败态是对于对手而言。也就是说，你可以通过一次操作使得对手必败，那么你就必胜。

那么当前有三包饼干时，Pretharp 必败，为什么？因为 Pretharp 可以将当前状态转换为还剩下一包或者两包饼干的状态，而这两个状态都是必胜态，且这个必胜态是对于对手而言，所以，如果你无论如何操作都会使得对手必胜，你就必败。

根据上面的例子我们可以得到 NP 状态的推导方法：

  • 如果当前状态可以通过一次操作变为 P 状态，那么当前状态为 N 状态。
  • 如果当前状态通过一次操作只能变为 N 状态，那么当前状态为 P 状态。

2.2 异或和世界观（Nim 游戏&台阶型 Nim 游戏）

Nim 游戏：

先来看一个很经典的问题——Nim 游戏：

桌子上有 nnn 堆饼干，第 iii 堆饼干有 aia_iai​ 包饼干。Pretharp 和 Mijacs 可以轮流从其中某一堆饼干中吃掉任意数目的饼干（但是不能不吃），如果没有饼干吃了就算输掉游戏。Pretharp 先手，当双方都采用最优状态时，问输赢。

我们根据定义来推导：

  • 我们必然可以将当前状态变成一个异或和为 000 的情况（也就是必败态）。为什么呢？假设当前异或和为 sss，我们只要让某一个数减去 ai−(ai⊕s)a_i - (a_i \oplus s)ai​−(ai​⊕s) 就可以使得 s=0s=0s=0。因为异或和不为 000，也就是在二进制下某一位上，所有数在这一位上为 111 的数的个数是奇数，所以 ai⊕s<sa_i \oplus s < sai​⊕s<s 的数必然存在，所以这一个操作必然合法。所以如果当前异或和不为 000，你一定可以将当前状态变为异或和为 000 的状态。

  • 因为当前状态是必败态，所以当前异或和为 000。之后你无论怎么操作之后异或和都不可能继续是 000。所以如果当前状态异或和为 000，你只能将当前状态转换为异或和不为 000 的状态。

根据上述推导，最终的必败态（所有饼干都被吃完了）的异或和为 000，而若当前态不为 000 你可以一直把异或和为 000 的状态留给对手，且对手只能把异或和不为 000 的状态留给你，直到某一次留给对手的状态为最终态，我们获胜。所以我们得出结论：

  • 若 ⨁i=1nai≠0\bigoplus\limits_{i=1}^{n}a_i \not= 0i=1⨁n​ai​=0（即 aia_iai​ 的异或和），则先手必胜（也就是初始状态是必胜态）。
  • 相反，先手必败（即初始状态为必败态）。

台阶型 Nim 游戏：

在进入下一个板块之前，我们还需要搞明白一个东西。引入问题：

在一个 n+1n+1n+1 级的台阶上（级数为 0,1,…,n0,1,\ldots,n0,1,…,n），第 iii（i∈[1,n]i \in [1,n]i∈[1,n]）级台阶有 aia_iai​ 包饼干放在上面。Pretharp 和 Mijacs 可以轮流选择一个在 111 到 nnn 的台阶（设其为第 xxx 级），将上面任意数量的饼干移动到下一级台阶上（即 x−1x-1x−1）。如果饼干到达了第 000 级台阶上就会被吃掉，无法操作者输掉游戏。Pretharp 先手，当双方都采用最优状态时，问输赢。

先假设 n=2n=2n=2。不难发现，第 111 级台阶上的饼干被移动了就相当于是 Nim 游戏中的饼干被吃掉了。可是如果将第 222 级台阶上的饼干移动就会使第 111 级台阶上的饼干数目增加，这是 Nim 游戏里没有的操作。但是聪明的你一定可以发现，对手在第 222 级台阶上移动了多少饼干下来，你就再在第 111 级台阶上移动多少饼干下去，让这次操作相当于没有操作，并且第 222 级台阶上的饼干数目有限，所以最终一定会被移动完。此时，答案就很显然了。

我们在此基础上拓展一下，对于 2n+12n+12n+1 和 2n+22n+22n+2 我们可以把它们看成一个整体：

  • 奇数台阶上的饼干数目就相当于 Nim 游戏中的一堆饼干。
  • 偶数台阶上就是一个饼干缓存区（也就是上一自然段所讲述的东西，到达了偶数台阶上的饼干只会临时在奇数饼干上路过一下，并不会停留在上面），即可以把到达了偶数台阶上的饼干看成到达了第 000 级台阶的饼干。

有了 Nim 游戏的铺垫，不难得出该游戏的结论：若 ⨁i∈[1,n],2∤iai≠0\bigoplus\limits_{i\in[1,n],2\nmid i} a_i \not= 0i∈[1,n],2∤i⨁​ai​=0（即所有奇数级台阶上的饼干数目的异或和），则先手必胜，反之亦然。

但重要的不是这个，而是我们要搞清楚：出现了不符合 Nim 游戏的操作，如果可以抵消且这种操作是有节制的，那我们可以无视这种操作。 (注：这一句话是笔者自己总结的，不保证 100%100\%100% 严谨，请不要当作结论使用)。

2.3 mex⁡\operatorname{mex}mex 运算

进入下一个板块前的最后东西。

mex⁡\operatorname{mex}mex 运算，即 minimum exclusion，定义：mex(S)=min⁡{x∣x∈N,x∉S}⁡\operatorname{mex(S)=\min\{x|x \in \mathbb{N},x\notin S\}}mex(S)=min{x∣x∈N,x∈/S}（即一个集合中最小的没出现过的非负整数），例如 mex({0,1,3})=2⁡\operatorname{mex(\{0,1,3\})=2}mex({0,1,3})=2。

3 SGSGSG 函数

3.1 有向图游戏

引入问题：

有 nnn 个有向无环连通图，每个有向图有且仅有一个起点（起点可以到达其所在图内任何一个点，且其所在图内任何点都无法到达）。现在，这 nnn 个图上，每个图的起点都放置了一包饼干，Pretharp 和 Mijacs 轮流选择一包饼干沿有向边移动，无法操作者输掉游戏，当双方都采用最优状态时，问输赢。

先定义什么是 SGSGSG 函数：SG⁡(x)=mex⁡({SG⁡(y1),SG⁡(y2),…,SG⁡(yk)})\operatorname{SG}(x)=\operatorname{mex}(\{\operatorname{SG}(y_1),\operatorname{SG}(y_2),\ldots,\operatorname{SG}(y_k)\})SG(x)=mex({SG(y1​),SG(y2​),…,SG(yk​)})，其中 xxx 的有向边分别连向 y1,y2,…,yky_1,y_2,\ldots,y_ky1​,y2​,…,yk​。为了方便表述，下文所说的「xxx 的 SGSGSG 值」等价于「SG⁡(x)\operatorname{SG}(x)SG(x)」。

我们会发现一些很巧妙地性质：

  • 对于一个结点，它可以走到任何一个 SGSGSG 值更小的结点。
  • 当一个结点不能再走下去的时候，其 SGSGSG 值为 000。
  • 如果一个结点走一步之后，其 SGSGSG 值变大，一定可以再走一步使其 SGSGSG 值变回原来的值。

我们换个说法，对于结点 xxx：

  • 它有 SG{x}⁡\operatorname{SG\{x\}}SG{x} 个饼干，并且你可以一次拿掉任意数量的饼干。
  • 当你不能再在这个结点拿饼干的时候，其饼干数目为 000。
  • 如果对手把饼干数目变多了，你可以把饼干数目变回来。

如果我们在一个图中，把每个结点看成一个状态，起点是初始状态……那一整个有向图就可以抽象为 Nim 游戏中的一堆石子。如果这 nnn 个有向图的起点分别是 u1,u2,…,unu_1,u_2,\ldots,u_nu1​,u2​,…,un​，如果 ⨁i=1nSG⁡(ui)≠0\bigoplus\limits_{i=1}^n\operatorname{SG}(u_i) \not= 0i=1⨁n​SG(ui​)=0（即每个有向图的起点的 SGSGSG 值的异或和）先手必胜，反之亦然。

在后续的大部分问题中，我们依然可以把每个子问题看作一个图，每一个图中的结点就是这个子问题所能触及的状态（就像流程图一样）。这只是 SGSGSG 函数最基础的运用，请读者务必将此模型完全理解。
